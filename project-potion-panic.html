Potion Panic

Multiplayer Unreal Engine capstone project.

Unreal Engine 5 C++ Multiplayer UI Systems

Overview

Potion Panic is a PC game that blends chaotic party gameplay with the simulation of running an apothecary shop. In each round, players must quickly juggle tasks to brew potions requested by customers within a limited time. As the game progresses, new ingredients and more complex steps are introduced, increasing the challenge while players strive to maximize their score.

What I Worked On

Gameplay Systems

  • Built components surrounding core gameplay loop including potion brewing, request handling, scoring, and round timers
  • Designed potion recipe system using structs and data tables for scalable ingredient management
  • Implemented request lifecycle (generation, timers, validation, and completion logic)
  • Added progression systems including level data, difficulty scaling, and ingredient variations
  • Developed GameMode, GameState, and PlayerState systems to control game flow
  • Introduced currency rewards, high score tracking, and end-of-round ranking

UI / UX

  • Designed and implemented main menu, HUD, pause menu, and end-of-round screens
  • Built dynamic UI for request tracking, timers, score display, and ingredient feedback
  • Added settings menus (graphics, audio) with persistent user preferences
  • Created save/load UI with multiple slots, deletion, and confirmation flows
  • Improved UI performance and stability by reducing redundant updates
  • Enhanced UX with visual feedback, icons, and layout refinements

Multiplayer & Networking

  • Implemented multiplayer support with synchronized GameState and PlayerState
  • Optimized server-client communication for UI and gameplay updates
  • Resolved replication issues to ensure consistent client state
  • Built lobby/ready system with player count display and controlled round start
  • Fixed multiplayer bugs including duplicate requests and level loading issues

Architecture & Systems Design

  • Established scalable project structure and core C++ systems in Unreal Engine
  • Developed reusable systems for UI, gameplay logic, and data handling
  • Transitioned hardcoded logic to data-driven systems using data tables
  • Improved maintainability by consolidating systems and reducing duplication
  • Implemented enums and structured state handling for cleaner flow control

Persistence & Progression

  • Built save game system with multiple slots and persistent data
  • Implemented level select and progression tied to save data
  • Ensured proper loading of user settings and game state across sessions

Tools, Debugging & Optimization

  • Created debugging tools and cheat systems for testing
  • Resolved crashes, memory issues, and edge cases
  • Tuned gameplay values for balance and playtesting
